API

Above you can check the API used in this application in case you want to user our service to make your own Educational Space Invaders Version.

Hack, Mod, Improve =D
Create User
"/createUser/{nickname}/{password}"
  • nickname - user's nickname
  • password - user's password
this service will get the nickname and create a user on the database
Login
"/login/{nickname}/{password}"
  • nickname - user's nickname
  • password - user's password
This service will get the nickname and password and return a hashkey if the user is logged, this hashkey will be used to user authentication on the other services
Get the from a specific game
"/GetGameData/{hash}/{GameID}"
  • hash - user's hash identifier, provided by login service
  • GameID - GameID of the game]
Service that response a JSON with all data of the provided GameID. Used for editor
Get all the games available
"/GetGameList/{hash}/{mode}" "/GetAllGamesList/"
  • hash - user's hash identifier, provided by login service
  • mode - 'all' for all games available, 'mine' for only the user's created games
The GameData of a specific gameid this service also works without the hash, but if the game is closed for public it wont work (use it -for the game)
Get all images availables (for gallery)
"/GetImageList/{hash}"
  • hash - user's hash identifier, provided by login service
get all images uploaded by the user (for gallery creation and reuse of sprites)
Delete an Asset from a game
"/deleteAsset/{hash}/{assetId}"
  • hash - user's hash identifier, provided by login service
  • assetID - ID of the asset to be deleted
This service delete a specific asset from a game. The image of the asset is not deleted only the reference of the image to a specific Game.
Delete a Game
"/deleteGame/{hash}/{gameID}"
  • hash - user's hash identifier, provided by login service
  • gameID - Id of the game to be deleted
This service delete the game reference and all it's assets. The Image file of the asset will be kept.
Create a new Game
"/createNewGame/{hash}/"
  • hash - user's hash identifier, provided by login service
This service create an empty game
Update a Game
"/updateGame/{hash}/{gameID}/{gameName}/{gameDescription}"
  • hash - user's hash identifier, provided by login service
  • gameID - ID of the game which data wll be modified
  • gameName - The new game name to be used
  • gameDescription - the new game description to be used
This service update the meta data of the game.
Update or Create an Asset
"/updateAsset/{hash}/{gameID}/{assetId}/{assetType}/{assetText}/{answerId}/{imageFileId}"
  • hash - user's hash identifier, provided by login service
  • gameID - Game ID which the asset will be associated
  • assetID - the asset ID that will be modified
  • assetType - the asset Type(enemy, player, background or answer) of the asset that will be created or modified
  • assetText - the asset Text of the asset that will be created or modified
  • answerID - set an asset as answer (used for enemy asset type only)
  • imageFieldID - set this ID as the image for that asset. (used for gallery)
This is the main asset function. The hash and the gameID are obligatory. The other ones are optional, to avoid changing the information add a -1 to the field, this will indicate that the data will not be replaced To upload a file just add a formdata and send it as POST. The service will answer a JSON with the data of this asset with its ID, Url and other information
